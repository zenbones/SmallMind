[[overview]]
== Overview

The `smallmind-nutsnbolts` module is the foundational utility jar that every
other module in the SmallMind repository builds on. It is a single artifact,
`smallmind-nutsnbolts`, that collects twenty-eight small, focused subpackages
of general-purpose Java infrastructure: command-line parsing, thread-bound
context stacks, CSV and XML processing, cryptography, layout calculation,
classloader control, property expansion, a resource abstraction, time
conversions, bag and bloom-filter collections, retry, e-mail, NTP, SSL,
servlet and SOAP helpers, and more.

None of these subpackages is a framework in its own right. Each fills a gap
in either the JDK or the Jakarta/Spring surface, and each is designed to be
usable in isolation, without pulling in the others. The package is named
"Nuts'n'Bolts" because that is what it is — small, durable, reusable parts
that other modules bolt together. Most other SmallMind jars depend on
`nutsnbolts` at compile time; very few things in this repository build
without it.

=== Why this package exists

It's generally useful to accumulate a common set of utility classes:
a GNU-style command-line parser, a thread-local stack of "current caller"
information, converters between `Date` / `Calendar` / `ZonedDateTime`, a
Base64 codec that pre-dates `java.util.Base64`, a classloader that can
exclude or gate specific paths, a property placeholder resolver with
encryption, a tiny retry wrapper, a bag, a bloom filter, JAXB adapters for
the `java.time` types that JAXB still does not marshal out of the box, a
helper that computes SHA-256 without fifteen lines of
`NoSuchAlgorithmException` handling, a classpath-aware FreeMarker loader,
and roughly another two dozen pieces in that vein.

`smallmind-nutsnbolts` is that set, written once, maintained in one place,
and reused across every other SmallMind artifact. It exists so that:

* An application can depend on a single jar to get all of the common
  building blocks, instead of pulling in fragments of Apache Commons,
  Guava, Spring, and Bouncy Castle solely to assemble a working baseline.
* Every Jakarta, Spring, Bouncy Castle, FreeMarker, and Commons-Net
  dependency is declared *optional*, so a consumer that does not use, for
  example, the e-mail or NTP subpackage does not pay for the corresponding
  jakarta.mail or commons-net transitive closure.
* Other SmallMind modules — `smallmind-batch`, `smallmind-bayeux`,
  `smallmind-liquibase`, `smallmind-file`, `smallmind-kafka`, the whole
  `web` stack — have a shared vocabulary for resources, contexts,
  properties, and I/O primitives.

=== What this package is not

* Not a competitor to Apache Commons, Guava, or Spring's utility surfaces.
  Where those libraries provide what is needed, nutsnbolts does not
  reimplement them. The subpackages here exist because the repository
  needed a specific semantics that none of those libraries provide out of
  the box, or because the repository needed to avoid the transitive
  footprint.
* Not a dependency-injection framework. The `inject` subpackage is a
  single marker annotation plus an AspectJ-driven lazy-field helper, not a
  container.
* Not a web or HTTP framework. The `http` subpackage wraps
  `HttpURLConnection` for simple request/response cases; full HTTP clients
  live elsewhere.
* Not a cryptography library. The `security` subpackage is a convenience
  surface over JCE and Bouncy Castle; callers who need specific algorithm
  modes or hardware-backed keys should go to those providers directly.

[[installation]]
== Installation

Add the artifact to your Maven build. All third-party couplings are marked
`optional` in the nutsnbolts POM, so the transitive surface is minimal:

[source,xml]
----
<dependency>
  <groupId>org.smallmind</groupId>
  <artifactId>smallmind-nutsnbolts</artifactId>
  <version>${project.version}</version>
</dependency>
----

[[conceptual-model]]
== Conceptual Model

A small number of cross-cutting concepts show up throughout the package.
Understanding them first makes every subpackage easier to read.

*Resource*::
    A uniform, read-only abstraction for addressable content. Every
    `Resource` has an identifier in `scheme:path` form (`classpath:config.xml`,
    `file:/etc/app.conf`, `jar:...`), and implementations in other SmallMind
    modules typically consume this interface rather than raw `File` or `URL`.
    See <<pkg-resource>>.

*Context stack*::
    A per-type, per-thread stack of objects implementing the `Context`
    marker. Pushing a context associates a value with the thread *and with
    every child thread spawned later* (the context map is inheritable).
    Pop balances push. This is how request identity, caller identity, and
    transaction metadata are propagated through SmallMind services
    without polluting method signatures. See <<pkg-context>>.

*PropertyExpander*::
    A configurable `${...}` placeholder engine that consults a map, system
    properties, and environment variables in a declared order, with
    optional decryption of values marked as encrypted. This engine backs
    the Spring `PropertyPlaceholderConfigurer` in this package and is used
    stand-alone by modules such as `ansible` and `spark` for template
    interpolation. See <<pkg-property>>.

*Closeable, idempotent primitives*::
    I/O helpers in this package are intentionally closable and safe to
    close twice. `ByteArrayIOStream`, `CircularBufferIOStream`, and the
    segmented buffers all treat `close` as a one-way latch, so
    try-with-resources usage is the norm.

*Optional over required dependencies*::
    Every Jakarta, Spring, FreeMarker, and Bouncy Castle dependency is
    *optional* in the POM. The subpackages that depend on them fail fast
    at class-load time (via `NoClassDefFoundError`) when a consumer
    exercises a code path whose optional dependency is missing. A
    consumer that does not use that code path experiences nothing.

[[pkg-command]]
=== command — GNU-style CLI parsing

*Packages:* `org.smallmind.nutsnbolts.command` (`CommandLineParser`,
`OptionSet`, `CommandLineException`) and
`org.smallmind.nutsnbolts.command.template` (`Template`, `Option`,
`Argument`, `ArgumentType`, `EnumeratedArgument`, `NoneArgument`,
`SingleArgument`)

Parses `String[] args` against a `Template` of options, returning an
`OptionSet` that callers query by option name or single-character flag.
Supports long options (`--verbose`), short flags (`-v`), arguments of
type `NONE`, `SINGLE`, `MULTIPLE`, `LIST`, and `ENUMERATED`, optional
child-option dependencies, and an opt-in "undeclared trailing arguments"
mode for positional tokens.

*Why it exists.* Rolling a CLI parser by hand is a recurring source of
bugs (unclosed quotes, missing argument validation, enumerated-value
typos). This parser is small, has no transitive dependencies, and is
configurable either programmatically via `Template`/`Option` objects or
declaratively via the shipped XSD at
`src/main/resources/xsd/org/smallmind/nutsnbolts/smallmind-command-1.0.xsd`.

*Key types.*

`CommandLineParser`::
    Static entry point. `parseCommands(Template, String[])` rejects any
    token that is not a declared option; the three-argument overload with
    `allowUndeclared = true` collects unrecognised trailing tokens as
    remaining arguments.

`Template`::
    Declares the set of valid `Option`s. Options can nest other options
    as dependents; a dependent may only be used when its parent is present.

`Option`, `Argument`, `ArgumentType`, `EnumeratedArgument`::
    The declarative building blocks; `ArgumentType` is the enum of
    `NONE`, `SINGLE`, `MULTIPLE`, `LIST`, `ENUMERATED`.

`OptionSet`::
    The parsed result. `containsOption(name)`, `getArgument(name)`,
    `getArguments(name)`, `getRemaining()` for undeclared trailing
    tokens.

`CommandLineException`::
    Thrown on unknown options, missing required options, or dependent
    options used without their parent.

*Example.*

[source,java]
----
Template template = new Template("myapp",
  new Option("verbose", 'v', false, NoneArgument.instance()),
  new Option("input",   'i', true,  new SingleArgument("path")),          // <1>
  new Option("level",   'l', false,
    new EnumeratedArgument(new String[] {"low", "medium", "high"}))
);

OptionSet options = CommandLineParser.parseCommands(template, args);     // <2>

boolean verbose = options.containsOption("verbose");
String input    = options.getArgument("input");
String level    = options.containsOption("level") ? options.getArgument("level") : "medium";
----
<1> The third argument to `Option` is the `required` flag. The parser
    throws `CommandLineException` if a required option is absent.
<2> `parseCommands` validates the entire array before returning. It never
    partially applies options.

[[pkg-context]]
=== context — thread-bound context stacks

*Package:* `org.smallmind.nutsnbolts.context`

Maintains a per-type, per-thread stack of objects implementing the
`Context` marker. The underlying map is an `InheritableThreadLocal`, so
a child thread inherits a *snapshot* (a new `LinkedList` copy) of the
parent's stack for every registered context type.

*Why it exists.* Request identity, caller identity, and other
"ambient" values must travel with a thread without appearing in every
method signature. Java's plain `ThreadLocal` handles the single-value
case but loses history; nested push/pop semantics are what most
real services actually need. The `ExpectedContexts` annotation lets
framework glue verify that required contexts are present at a given
method boundary.

*Key types.*

`Context`::
    Marker interface. Implementations must be identifiable by their
    concrete class; the class is used as the map key.

`ContextFactory`::
    Static stack operations: `pushContext`, `popContext`,
    `getContext`, `removeContext`, `exists`, `sizeFor`, plus
    `importContextTrace` / `exportContextTrace` for serialising the
    current thread's stack across an asynchronous boundary.

`ExpectedContexts`::
    Method-level annotation declaring which `Context` types must be
    active when the method runs. `ContextFactory.filterContextsOn(method)`
    validates and collects them.

`LifecycleAware`::
    Optional lifecycle hook (push/pop callbacks) for contexts that need
    to perform setup or teardown around their stack lifetime.

`ContextException`::
    Unchecked, thrown when `ExpectedContexts` expectations are not met.

*Example.*

[source,java]
----
public record CallerContext(String userId, String tenantId) implements Context {}

try {
  ContextFactory.pushContext(new CallerContext("u-42", "t-7"));       // <1>
  processRequest();
} finally {
  ContextFactory.popContext(CallerContext.class);                     // <2>
}

// Deep in the call stack — no parameter plumbing:
CallerContext caller = ContextFactory.getContext(CallerContext.class);
----
<1> Push into this thread's stack for the concrete `CallerContext` key.
<2> Always pop in a `finally`. Failing to pop is the single most common
    bug; the context survives the method and pollutes subsequent work
    on the same thread.

[[pkg-resource]]
=== resource — uniform addressable content

*Package:* `org.smallmind.nutsnbolts.resource`

A small interface — `Resource` — that exposes a scheme, a path, an
identifier, and an `InputStream`, plus concrete implementations for
`file:`, `classpath:`, `jar:`, and URL-backed resources, and a
`ResourceParser` that constructs them from identifier strings of the
form `scheme:path`.

*Why it exists.* Configuration, templates, and data files come from
multiple places: filesystem paths, classpath JAR entries, deployment
overlays, remote URLs. Callers should not special-case each one. This
abstraction keeps `Resource` usage uniform throughout the rest of the
SmallMind codebase, and combined with `ResourceFactory` it integrates
cleanly with Spring as a `FactoryBean<Resource>`.

*Key types.*

`Resource`::
    The interface. `getIdentifier()`, `getScheme()`, `getPath()`, and
    `getInputStream()`. The input stream may return `null` if the
    backing store does not locate the content (rather than throwing),
    and the scheme is suitable as a map key.

`ResourceParser`, `ResourceGenerator`::
    Parses `scheme:path` strings into `Resource` instances. A
    `ResourceGenerator` knows which schemes it can produce; the parser
    maps each scheme to exactly one generator.

`ResourceFactory`::
    A Spring `FactoryBean<Resource>` that reads a `name` property
    (a resource identifier string) and vends the parsed `Resource`
    as a singleton.

`FileResource`, `ClasspathResource`, `JarResource`::
    Concrete implementations. Instantiate directly when you know the
    backing store; otherwise go through the parser.

`ResourceException`::
    Thrown when a resource cannot be opened or parsed.

*Example.*

[source,java]
----
ResourceParser parser = new ResourceParser(new ResourceTypeResourceGenerator());

Resource cfg = parser.parseResource("classpath:config/app.yml");  // <1>
try (InputStream in = cfg.getInputStream()) {
  // ... load properties
}
----
<1> Or `file:/etc/app/app.yml`, or `jar:...`. The rest of the code does
    not change.

[[pkg-property]]
=== property — placeholder expansion

*Package:* `org.smallmind.nutsnbolts.property`

`PropertyExpander` resolves `${...}` (or a custom delimiter) placeholders
in a string by consulting a caller-supplied map, system properties, and
environment variables, in a declared order. It detects circular
references, tolerates nested placeholders, and supports optional
decryption of values via a pluggable `Decryptor` wrapped in a
`PropertyClosure`.

*Why it exists.* Externalised configuration often has placeholders. Some
values are encrypted at rest; some must be resolved differently when a
system property shadows a file value. A single expander with explicit
resolution order, optional decryption, and strict-vs-tolerant behaviour
is simpler to reason about than assembling one ad hoc per application.

*Key types.*

`PropertyExpander`::
    Entry point. Construct with a `PropertyClosure`, an
    `ignoreUnresolvableProperties` flag, a `SystemPropertyMode`
    (`OVERRIDE` or `FALLBACK`), and a `searchSystemEnvironment` flag.
    Call `expand(String)` or `expand(String, Map)`.

`PropertyClosure`::
    Delimiter configuration (default `${` / `}`) plus an optional
    `Decryptor` that is invoked when a placeholder value carries an
    encryption marker.

`PropertyExpanderException`::
    Unresolvable placeholder, unclosed prefix, or circular reference.

`SystemPropertyMode`::
    Enum with `OVERRIDE` (system properties take precedence over the
    caller map) and `FALLBACK` (system properties are only consulted
    when the caller map lacks the key).

*Example.*

[source,java]
----
PropertyExpander expander = new PropertyExpander(
  new PropertyClosure(),                // default ${...} delimiter
  false,                                // strict: throw on unresolved
  SystemPropertyMode.FALLBACK,          // caller map wins
  true                                  // also consult environment
);

Map<String, Object> ctx = Map.of("service", "api", "env", "prod");
String url = expander.expand("https://${service}.${env}.example.com");  // <1>
----
<1> Placeholders are resolved left-to-right; nested placeholders such as
    `${prefix.${region}}` are supported. Unknown keys throw because
    `ignoreUnresolvableProperties` is `false`.

[[pkg-io]]
=== io — streams, buffers, and path helpers

*Package:* `org.smallmind.nutsnbolts.io`

Segmented in-memory streams and buffers that outperform
`ByteArrayOutputStream` for large payloads, a circular buffer for
bounded producer/consumer queues, and path utilities for recursive
copy/configuration.

*Why it exists.* `ByteArrayOutputStream` is a single contiguous byte
array; writing a 100 MB payload means the backing array reallocates
and copies many times. The segmented `ByteArrayIOBuffer` grows by
appending fixed-size segments instead, avoiding quadratic
reallocation. `ByteArrayIOStream` additionally lets the reader and
the writer advance independently against the same buffer.

*Key types.*

`ByteArrayIOStream`::
    A segmented in-memory stream with separate `ByteArrayInputStream`
    and `ByteArrayOutputStream` views. Constructors allow choosing the
    segment size (default 1 024 bytes) or reusing an existing
    `ByteArrayIOBuffer`. `close()` is a one-way latch that poisons both
    views.

`ByteArrayIOBuffer`::
    The underlying segmented buffer. Useful on its own when you need
    reference sharing between two `ByteArrayIOStream` instances.

`CircularBuffer`, `CircularBufferIOStream`::
    Fixed-size ring buffer for bounded producer/consumer patterns, and
    the stream view over it.

`CloseListener`, `CloseEvent`, `CloseableProxyFactory`::
    Callback mechanism to observe `close()` on resources whose
    lifecycle is managed elsewhere.

`ByteBufferInputStream`, `MultiOutputStream`, `MarkableInputStream`::
    `InputStream` view over a `ByteBuffer`'s remaining bytes, an
    `OutputStream` that fans every write out to a fixed array of
    delegate streams, and a wrapper that adds `mark` / `reset` support
    to any `InputStream` that doesn't natively offer it.

`FileUtility`, `PathUtility`, `CopyTreeConfigurationBuilder`,
`DeleteTreeConfigurationBuilder`::
    Recursive tree copy, recursive delete (with optional
    `DirectoryNotEmptyException` propagation), `isDirectoryEmpty`, and
    builders for copy/delete configuration including per-path filters
    and directory-creation attributes.

`RegExTranslator`, `ExtensionFileFilter`, `WildcardFileNameFileFilter`,
`WildcardFileNamePathFilter`, `PathFilter`, `TimeFilter`::
    Glob-to-regex translation (where `*` and `?` do not cross `/`,
    while `**` does), file/path filters that consume those patterns,
    and a `PathFilter` that accepts paths whose last-modified time
    satisfies a `time.TimeArithmetic` predicate.

*Example.*

[source,java]
----
try (ByteArrayIOStream pipe = new ByteArrayIOStream(8 * 1024)) {
  OutputStream out = pipe.asOutputStream();                       // <1>
  out.write("hello".getBytes(StandardCharsets.UTF_8));

  InputStream in = pipe.asInputStream();                          // <2>
  byte[] readBack = in.readAllBytes();
}
----
<1> The output view and the input view share the underlying segmented
    buffer; there is no copy.
<2> The input view maintains its own read cursor and supports
    mark/reset.

[[pkg-csv]]
=== csv — event-driven CSV parsing

*Package:* `org.smallmind.nutsnbolts.csv`

`CSVParser` reads a stream line-by-line, emits `startDocument`, one
`handleFields(String[])` per record, and `endDocument` to a
`CSVLineHandler`. The low-level `CSVReader` handles quoting, embedded
delimiters, multi-line quoted fields, and `""` as the escape for a
literal quote inside a quoted field. Optional flags for "skip header"
and "trim fields".

*Why it exists.* CSV is ubiquitous but error-prone (quoted fields
containing commas, embedded newlines, escape characters). An
event-driven handler model processes arbitrarily large files without
materialising them in memory.

*Example.*

[source,java]
----
CSVParser parser = new CSVParser();
parser.setSkipHeader(true);                                       // <1>
parser.setTrimFields(true);
parser.setLineHandler(new CSVLineHandler() {
  @Override public void startDocument() {}
  @Override public void handleFields(String[] fields) {
    // fields[0], fields[1], ...
  }
  @Override public void endDocument() {}
});

parser.parse(Files.newInputStream(Paths.get("data.csv")));
----
<1> Skip the header row. Otherwise the first `handleFields` call delivers
    the column titles.

[[pkg-xml-json]]
=== xml, json — XML utilities and JAXB adapters

*Packages:* `org.smallmind.nutsnbolts.xml`, `org.smallmind.nutsnbolts.json`

`xml` provides DOM serialisation, XML escaping, and pluggable
`ProtocolResolver`s for custom URI schemes during SAX/DOM parsing
(useful for resolving DTDs and XSDs without hitting the network).

marshal out of the box: `LocalDateTime`, `ZonedDateTime`, `Currency`,

These two subpackages hold the shared implementations.

*Example — DOM to string.*

[source,java]
----
String xml = XMLUtility.toString(document, true);                 // <1>
----
<1> Indents using three spaces per nesting level when the boolean flag
    is `true`; the single-argument overload emits an unindented form.

    in XML content (it replaces `&`, `<`, and `>`).

[source,java]

@XmlElement

[[pkg-freemarker]]
=== freemarker — template source adapters

*Package:* `org.smallmind.nutsnbolts.freemarker`

`ClassPathTemplateLoader`, `FileSystemTemplateLoader`, and
`ResourceTemplateLoader` plug into FreeMarker's `TemplateLoader`
contract so templates can come from the classpath, a filesystem
path, or a SmallMind `Resource`. The matching `*TemplateSource`
classes wrap the underlying handle (input stream, file path, or
`Resource`) and are what the loader hands back to FreeMarker on each
`findTemplateSource` call.

*Why it exists.* FreeMarker ships loaders, but they assume certain
deployment layouts. A classpath loader that searches with the
context classloader, respects JAR entries, and caches last-modified
times is a routine need.

*Key constructors.*

`ClassPathTemplateLoader`::
    Three constructor shapes: no-argument (uses the context
    classloader and absolute resource names), single-`ClassLoader`
    (custom classloader, absolute names), or single-`Class` /
    `Class + boolean relative`. With `relative = true` the loader
    prefixes every requested name with the anchor class's package
    path, so a template colocated with its consumer can be loaded by
    its short name.

`FileSystemTemplateLoader`::
    Either no-argument (template names are treated as absolute file
    paths, with a leading slash inserted because FreeMarker strips
    them as a security measure) or with a base `Path` that template
    names are resolved against. `setBasePath` switches the root at
    runtime.

[[pkg-email]]
=== email — Jakarta Mail wrapper

*Package:* `org.smallmind.nutsnbolts.email`

`Postman` constructs Jakarta Mail `Session`s over SMTP or SMTPS, with
or without authentication, and transmits `Mail` descriptors.
FreeMarker bodies are supported out of the box: passing a
non-`null` `HashMap<String, Object>` to
`Postman.send(Mail, HashMap)` compiles the message body as a
FreeMarker template and caches the compiled template by the
SHA-256 of the source text.

*Why it exists.* Jakarta Mail is a decade of verbose boilerplate
per message. `Mail` is a mutable value object for addressing,
subject, body (`String` or `Reader`), plaintext/HTML flag, and
attachments; `Postman` turns it into a `MimeMessage` and sends it.

*Key types.*

`Postman`::
    Configured with host, port, `Authentication`, and a "secure"
    flag that selects SMTPS (STARTTLS). `send(Mail)` or
    `send(Mail, HashMap<String, Object>)` for FreeMarker interpolation.

`Mail`::
    Addressing, subject, body, attachments. Multiple constructors
    for common shapes; setters for everything else. `setHtml(true)`
    emits `text/html`.

`Authentication`, `AuthType`::
    Authentication descriptor and the enum of supported strategies.
    Use the shared `Authentication.NONE` constant or construct via the
    no-argument constructor and populate `setType(AuthType)` /
    `setData(String[])`.

`Protocol`::
    `SMTP` / `SMTPS`. Used internally by `Postman` to build the
    `Session`.

`MailDeliveryException`::
    Wraps any construction or transport failure.

*Example.*

[source,java]
----
Authentication auth = new Authentication();
auth.setType(AuthType.LOGIN);
auth.setData(new String[] {"app@example.com", "secret"});

Postman postman = new Postman(
  "smtp.example.com", 587,
  auth,
  true                                                             // <1>
);

Mail mail = new Mail(
  "app@example.com",
  "alerts@example.com",
  "Build ${buildId} failed on ${host}"                              // <2>
);
mail.setSubject("CI failure");

HashMap<String, Object> model = new HashMap<>();                    // <3>
model.put("buildId", "4217");
model.put("host", "ci-3");
postman.send(mail, model);
----
<1> STARTTLS/SMTPS.
<2> Body is a FreeMarker template because the interpolation map in
    `send` is non-null.
<3> Compiled template is cached by SHA-256 of the body text; repeated
    sends of the same template recompile only once.

[[pkg-http-net-ntp]]
=== http, net, ntp — network utilities

*Packages:* `org.smallmind.nutsnbolts.http`,
`org.smallmind.nutsnbolts.net`, `org.smallmind.nutsnbolts.ntp`

`http` wraps `HttpURLConnection` for simple GET/PUT/POST/DELETE cases
(`HttpTransmitter`, `HttpPipe`, `HttpMethod`) and provides the common
codec utilities used during HTTP plumbing — `Base64Codec` (standard,
URL-safe, and custom-alphabet variants), `URLCodec` (RFC-3986

`application/x-www-form-urlencoded` query strings, and `HTMLUtility`

ordering puts `10.0.0.1` before `2.0.0.1`).

the computed clock offset in milliseconds. Use it to detect local
clock skew before making time-sensitive decisions.

*Example — NTP offset.*

[source,java]
----
NTPTime ntp = new NTPTime(new String[] {"pool.ntp.org", "time.google.com"});
long offsetMillis = ntp.getOffset(5_000);                         // <1>

[[pkg-security-ssl]]
=== security, ssl — cryptography and TLS

*Packages:* `org.smallmind.nutsnbolts.security`,
`org.smallmind.nutsnbolts.ssl`

`security` is a convenience layer over JCE with type-safe algorithm
enums. `EncryptionUtility` provides static methods for hashing,
symmetric and asymmetric signing/verification, symmetric encryption
and decryption, key-pair generation, and serialisation via `KeyRep`.
`AsymmetricKeyGenerator`, `CertificateUtility`, and the `x509` /
`kms` subpackages handle higher-level key-management scenarios —
these require Bouncy Castle on the classpath and, in some cases,
the provider registered.

`ssl` provides `KeyStoreUtility`, `TrustManagerUtility`, and a set of
*test-only* permissive implementations — `NaiveTrustManager`,
`NaiveHostNameVerifier`, `NaiveSSLSocketFactory` — for cases where TLS
must be short-circuited in an isolated test environment.

*Why they exist.* JCE boilerplate is noisy and easy to get subtly
wrong (wrong cipher mode, missing padding, missing IV). Enum-driven
algorithm selection (`HashAlgorithm.SHA_256`,
`SymmetricSigningAlgorithm.*`, `AsymmetricSigningAlgorithm.*`)
collapses the typo surface, and SHA-256 in one line instead of five
is why this utility exists.

*Key types.*

`EncryptionUtility`::
    Static: `hash`, `sign`, `verify`, `encrypt`, `decrypt`,
    `generateKeyPair`, `serializeKey`, `deserializeKey`, and helpers
    like `convertToBlock` for PEM-style line wrapping.

`HashAlgorithm`::
    Enum of supported message digests (`MD5`, `SHA_1`, `SHA_256`,
    `SHA_512`).

`HexCodec`::
    Lowercase hexadecimal encoding (with zero-padding for byte values
    below `0x10`) and decoding (accepting both upper and lower case,
    rejecting odd-length and non-hex input). Used by `SnowflakeId` and
    available standalone.

`SymmetricAlgorithm`, `AsymmetricAlgorithm`::
    Enums for cipher selection.

`SymmetricSigningAlgorithm`, `AsymmetricSigningAlgorithm`::
    Interfaces that concrete algorithm enums implement.

`HMACSigningAlgorithm`, `ECDSASigningAlgorithm`, `RSASigningAlgorithm`::
    Enum implementations providing the signing algorithm constants.
    `HMACSigningAlgorithm.HMAC_SHA_256` is the one callers reach for
    most often.

`AsymmetricKeyGenerator`::
    Generates RSA/EC key pairs; `AsymmetricKeySpec` and
    `AsymmetricKeyType` describe the requested shape.

`CertificateUtility`::
    X.509 certificate helpers (requires Bouncy Castle).

`SecurityProviderInitializingBean`::
    Spring `InitializingBean` that registers a
    `SecurityProvider` with the JVM during context startup.

`KeyStoreUtility`, `TrustManagerUtility`::
    Build `KeyStore`, `KeyManagerFactory`, and `TrustManagerFactory`
    instances from configured material.

`NaiveTrustManager`, `NaiveHostNameVerifier`, `NaiveSSLSocketFactory`::
    Skip certificate and hostname validation. Test-only.

[IMPORTANT]
====
`Naive*` in the `ssl` subpackage name is not ornamental. These
classes accept every certificate. Do not use them in production, and
do not register them as the default `HostnameVerifier` or
`SSLSocketFactory` in long-running JVMs.
====

*Example — SHA-256.*

[source,java]
----
byte[] digest = EncryptionUtility.hash(
  HashAlgorithm.SHA_256,
  payload.getBytes(StandardCharsets.UTF_8)
);
----

*Example — HMAC sign/verify.*

[source,java]
----
byte[] mac = EncryptionUtility.sign(
  HMACSigningAlgorithm.HMAC_SHA_256, sharedKey, message
);

boolean ok = EncryptionUtility.verify(
  HMACSigningAlgorithm.HMAC_SHA_256, sharedKey, message, mac
);
----

[[pkg-time]]
=== time — date and time conversions

*Package:* `org.smallmind.nutsnbolts.time`

`TimeUtility` converts legacy types (`Date`, `Calendar`, epoch
milliseconds, `LocalDateTime`) to `ZonedDateTime` and parses/formats
ISO-8601 strings in all four common shapes (date-only, local, offset,
zoned) using the corresponding `java.time.format.DateTimeFormatter`,
auto-detecting the shape from the input.

`Stint` is an immutable duration value (a non-negative `long` paired
with a `TimeUnit`); `StintUtility.convertToDouble` converts between
units with sub-unit precision preserved when going from finer to
coarser units. `TimeArithmetic` pairs a reference `ZonedDateTime` with
a `TimeOperation` (`BEFORE`, `BEFORE_OR_ON`, `ON_OR_AFTER`, `AFTER`)
to test arbitrary `Instant` values against a single reference point —
used by `io.TimeFilter` for filesystem tree walks. `Day` and `Month`
are enums; `CalendarUtility` provides integer-driven calendar
arithmetic (days in month, day of week via Zeller's congruence,
simplified `% 4` leap-year rule).

*Why it exists.* Java's `Date` → `java.time` migration is only mostly
done in real codebases. A single utility that turns any legacy value
into a `ZonedDateTime` — and back — cuts down the conversion boilerplate
dramatically.

*Example.*

[source,java]
----
ZonedDateTime now = TimeUtility.fromMilliseconds(System.currentTimeMillis());
ZonedDateTime parsed = TimeUtility.parse("2026-04-21T12:00:00Z");
String iso = TimeUtility.format(now);
----

[[pkg-retry]]
=== retry — retry with back-off

*Package:* `org.smallmind.nutsnbolts.retry`

`Retry.execute(retryCall, retries, delay, exponential)` runs the
supplied `RetryCall` on a fresh thread, joins it, and repeats up to
`retries` additional attempts with either a flat delay or `delay *
attemptNumber`. Returns `true` on success.

*Why it exists.* The code path is short but error-prone. Encapsulating
retry-with-back-off once avoids bespoke loops. Running each attempt
on a dedicated thread keeps a pathological blocking `RetryCall` from
spilling delay time onto the caller.

*Example.*

[source,java]
----
boolean ok = Retry.execute(
  () -> { httpClient.ping("https://api.example.com/health"); },  // <1>
  3,       // retries
  1_000L,  // 1-second base delay
  true     // exponential: 1s, 2s, 3s between attempts
);
----
<1> `RetryCall` is a functional interface that may throw `Throwable`.

[[pkg-lang-reflection-apt]]
=== lang, reflection, apt — classloaders and metaprogramming

*Packages:* `org.smallmind.nutsnbolts.lang`,
`org.smallmind.nutsnbolts.reflection`,
`org.smallmind.nutsnbolts.apt`

*`lang`* contains classloader infrastructure, exception helpers, and
the `AnnotationLiteral` proxy factory for synthesising annotation
instances at runtime:

`GatingClassLoader`::
    A `SecureClassLoader` that delegates class resolution through a
    chain of `ClassGate`s. Gates decide which resources a given child
    loader may see; a `ClasspathClassGate` accepts a list of
    filesystem or JAR paths. Optional grace-period logic compares
    last-modified timestamps to detect stale class definitions for
    reloadable extensions.

`ClassGate`, `ClasspathClassGate`, `ClassGateTicket`,
`ClassStreamTicket`, `ClassVerifier`::
    Gate SPI, the classpath-based default, and ASM-backed bytecode
    verification helpers.

`ClassLoaderAwareCache`::
    A cache whose entries are scoped to the classloader of the key
    type — entries vanish when the classloader is collected,
    preventing classloader leaks.

`AllPermissionsClassLoader`::
    A trivial child loader that grants every permission; audit-only.

`AnnotationLiteral`::
    Runtime factory for synthetic annotation proxies — useful for
    programmatic service discovery and metadata injection.

`FormattedException` / `FormattedRuntimeException`::
    Checked and unchecked exception base classes whose detail messages
    are built via `String.format(message, args)`. Subclasses across the
    repository (such as `CSVParseException`, `SpreadParserException`,
    `DotNotationException`, `MutationException`) inherit this pattern
    so callers can write
    `throw new FooException("Bad id(%s)", id)` instead of pre-formatting.

`WrappedException`::
    Adapts a checked exception into a `RuntimeException` for
    propagation through functional APIs, with a `convert(Class<E>)`
    accessor that casts the wrapped cause back to its original type.

`UnknownSwitchCaseException`::
    Convenience throw for default branches of exhaustive switches,
    naming the unhandled case.

`StackTraceAccumulator` / `ArrayStackTraceAccumulator` / `StackTraceUtility`::
    Pluggable sink for stack-trace lines plus a static
    `obtainStackTraceAsString(Throwable)` helper used by
    `http.HTMLUtility.convertThrowable` and elsewhere.

*`reflection`* provides field introspection (`FieldAccessor`,
`FieldUtility`), annotation filtering (`AnnotationFilter`), bytecode
debugging via ASM (`ClassInspector.trace`, `.asmify`), and getter
matching.

*`apt`* exposes `AptUtility`, a small set of methods for reading
`AnnotationMirror` values from the compile-time annotation-processor
environment. Used by SmallMind's own annotation processors.

*Example — gated classloader for a plugin.*

[source,java]
----
ClassGate gate = new ClasspathClassGate("plugins/acme-extension.jar");

GatingClassLoader loader = new GatingClassLoader(
  Thread.currentThread().getContextClassLoader(),
  0,                                                               // <1>
  gate
);

Class<?> entry = loader.loadClass("com.acme.ExtensionEntry");
// ... reflectively invoke an extension contract
----
<1> Grace period in seconds; a negative value disables staleness
    checking entirely.

[[pkg-layout]]
=== layout — platform-neutral layout engine

*Package:* `org.smallmind.nutsnbolts.layout`

A three-pass (minimum / preferred / maximum) measurement layout engine
with *serial* and *parallel* boxes, grow/shrink constraints,
alignment, justification, gaps, and bias (which axis is the "main"
one). Written without any specific GUI toolkit in mind — it operates
over the generic `ParaboxElement<T>` wrapper that a concrete toolkit
binds to its component type.

*Why it exists.* Layout code in Swing/JavaFX/web is the same math each
time: resolve minimum sizes bottom-up, spread surplus based on grow
weights, resolve alignment on the cross axis. Factoring that math
into a toolkit-agnostic engine lets SmallMind's `javafx` and desktop
modules share a single implementation.

*Key types.*

`ParaboxLayout`::
    The top-level layout coordinator. Constructed around a
    `ParaboxContainer` adapter for the host GUI toolkit. Produces
    `SerialBox` and `ParallelBox` instances via `serialBox(...)`
    and `parallelBox(...)` factory methods and exposes the
    horizontal and vertical root boxes via `setHorizontalBox` /
    `setVerticalBox`.

`Box<B>`::
    Abstract base for child boxes. Subclasses implement
    `doLayout`, `calculateMinimumMeasurement`,
    `calculatePreferredMeasurement`,
    `calculateMaximumMeasurement`.

`SerialBox`, `ParallelBox`::
    The two concrete box types. Serial arranges children along the
    main axis; parallel stacks children on top of each other
    against a shared cross axis.

`ParaboxContainer`, `ParaboxPlatform`::
    Adapters the host toolkit implements so the engine can ask it
    for child measurements and apply computed positions.

`ParaboxElement<T>`, `ComponentParaboxElement`, `BoxParaboxElement`::
    Wraps a toolkit component or a nested `Box` as a child of a
    layout box. Concrete toolkits subclass `ComponentParaboxElement`
    to adapt their widget type.

`Constraint`::
    Captures the grow and shrink weights for a child. Shared
    factories are `Constraint.immutable()` (rigid),
    `Constraint.expand()` (grow only), `Constraint.contract()`
    (shrink only), and `Constraint.stretch()` (both). Use
    `Constraint.create()` for a mutable variant, or
    `new Constraint(grow, shrink)` for explicit weights.

`Bias`::
    `HORIZONTAL` or `VERTICAL`. Drives which axis each calculation
    runs on.

`Alignment`, `Justification`, `Flow`, `Gap`, `Dimensionality`,
`Perimeter`, `LayoutTailor`::
    Supporting types.

*Sketch.*

[source,java]
----
ParaboxLayout layout = new ParaboxLayout(container);                  // <1>

SerialBox horizontal = layout.serialBox()
  .add(labelElement,  Constraint.immutable())
  .add(inputElement,  Constraint.stretch())
  .add(buttonElement, Constraint.immutable());

SerialBox vertical = layout.serialBox()
  .add(headerElement, Constraint.immutable())
  .add(bodyElement,   Constraint.expand());

layout.setHorizontalBox(horizontal);
layout.setVerticalBox(vertical);
----
<1> `container` is a `ParaboxContainer` supplied by the host toolkit
    (e.g. a Swing or JavaFX adapter).

[[pkg-util]]
=== util — collections and comparators

*Package:* `org.smallmind.nutsnbolts.util`

A toolkit of small, general-purpose utilities. The ones a consumer
is most likely to reach for:

`Bag<E>` / `HashBag<E>`::
    A multiset collection with duplicate counting. Adds and removes
    in multiples.

`BloomFilter`::
    Space-efficient probabilistic set (false positives possible, no
    false negatives).

`LRUMap<K,V>`::
    `LinkedHashMap`-backed bounded cache; insertion-order eviction by
    default, access-order via the two-arg `LRUMap(true, size)` constructor.

`AlphaNumericComparator`::
    Char-by-char sort where digits sort before letters and letters
    compare case-insensitively. Note that the comparison is per
    character, not per numeric run, so `"a10"` sorts before `"a2"`.

`SemanticVersionComparator`::
    Sorts dot-separated numeric version strings segment-by-segment as
    integers (so `"1.10.0"` correctly sorts after `"1.2.0"`). Null and
    empty inputs sort first.

`DotNotationComparator`::
    Comparator for dot-separated strings (paths, domain names, log
    categories) that compares each segment using
    `AlphaNumericComparator`. The `reversed` constructor flag changes
    segment iteration order to right-to-left, useful when the rightmost
    segment is the most significant key (such as the TLD of a domain
    name); it does *not* invert the sign of the comparison result.

`SnowflakeId`, `SnowflakeVariant`, `SnowflakeVersion`::
    Eighteen-byte identifiers combining a millisecond timestamp, the
    host MAC address (or random fallback), the JVM process id, and a
    per-millisecond counter. Supports hex, dotted, and compact
    alphanumeric encodings, with `Comparable` ordering by timestamp.

`Pair<L,R>`::
    Immutable two-element record. `Pair.of(a, b)` is the typical
    constructor.

`Tuple<K,V>`::
    Ordered, duplicate-allowing collection of key/value pairs with
    iterator support and fail-fast concurrent-modification detection.

`Counter`::
    Mutable `int` counter with pre- and post-variants of
    `incAndGet` / `getAndInc` / `decAndGet` / `getAndDec` /
    `addAndGet` / `getAndAdd` / `subtractAndGet` / `getAndSubtract`.
    Not thread-safe — for that, use `AtomicInteger`.

`Mutation<T,R>` / `MutationUtility`::
    A `Function`-like functional interface whose `apply` is allowed to
    throw checked exceptions. `MutationUtility` runs a `Mutation` over
    arrays, iterators, iterables, and collections to produce arrays,
    lists, sets, maps, or `Bag`s, wrapping any thrown exception in a
    `MutationException`.

`Option` / `Some` / `None`::
    Lightweight option-type helpers for codebases that prefer to model
    presence/absence explicitly without `java.util.Optional`.

`AnsiColor`::
    Terminal color escape sequences.

`Bytes`::
    Big-endian conversion between primitive `long` / `int` / `short`
    values and fixed-width byte arrays, used by `SnowflakeId` and
    elsewhere to pack numeric identifiers. For hex encoding see
    `HexCodec` in the `security` package.

`SelfDestructiveMap<K,V>`::
    A map whose entries expire after a configurable TTL; uses
    `SelfDestructiveKey` under the hood.

`NumberComparator`::
    Cross-type numeric comparator that normalises across `BigDecimal`,
    `BigInteger`, floating-point, and integral primitive wrappers
    before comparing. Throws `FormattedRuntimeException` when handed a
    `Number` subtype it does not recognise.

`DirectionalComparator<T>` / `NaturalDirectionalComparator<T>`::
    Abstract base that carries a sort `Direction` (`ASCENDING` /
    `DESCENDING`), and a concrete subclass that delegates to
    `Comparable.compareTo`.

`DotNotation`::
    Compiles a glob-style dotted pattern (segments joined by `.`, with
    `*` and `?` as wildcards) into a `Pattern`, and computes a bitmask
    value reflecting which segments matched. Rejects malformed input
    such as leading or trailing dots, consecutive dots, or wildcards
    mid-segment via `DotNotationException`.

`Spread` / `NumericSpread` / `AlphaSpread`::
    Expand comma-delimited range expressions such as `"1,3..5,9"` or
    `"a,c..e,z"` into arrays. `Spread` dispatches to the numeric or
    alphabetic variant based on the first character. Errors are
    surfaced as `SpreadParserException`.

`MagicHash`::
    XOR-shift hash mixer that reduces clustering and improves bit
    distribution for `int` hash codes. Useful when building custom
    hash tables over inputs whose natural hash code clusters in the
    low bits.

`MartianBase34`::
    Encodes non-negative `long` values into a 34-character custom
    alphabet, applying a per-`Group` additive mix constant. Three
    `Group` constants (`FIRST`, `SECOND`, `THIRD`) carry distinct mix
    values. Best used with explicit digit widths that comfortably
    accommodate the mix constant — auto-width selection may produce
    encoded strings whose decode does not round-trip cleanly.

`MacAddress` / `MacAddressException`::
    Reads the host MAC address from the first non-loopback network
    interface. Used by `SnowflakeId` to embed a machine identifier in
    every generated id.

`LocaleUtility`, `StringUtility`, `EnumUtility`, `ArrayUtility`::
    Assorted helpers. `EnumUtility.toEnumName` converts free-form text
    into upper snake-case enum constant names with configurable
    digit-prefix handling. `StringUtility` covers display/camel-case
    conversion, Java-identifier validation, and ellipsis trimming.
    `ArrayUtility` clones, concatenates, and linear-searches generic
    arrays. `LocaleUtility.forCountryISO3Code` looks up a `Locale` by
    its three-letter ISO country code.

*Example — LRU cache.*

[source,java]
----
LRUMap<String, User> cache = new LRUMap<>(1_000);                 // <1>
User user = cache.computeIfAbsent(userId, id -> repo.loadUser(id));
----
<1> Fixed capacity; the oldest entry by insertion order is evicted when full.

[[pkg-inject]]
=== inject — `@Service` marker and lazy fields

*Package:* `org.smallmind.nutsnbolts.inject`

`@Service`::
    A marker annotation for classes that participate in custom
    service-discovery scans. SmallMind framework code (and
    applications built on top of it) uses this marker to locate
    components without pulling in a specific DI container.

`@LazyField`, `LazyFieldAspect`, `LazyError`::
    An AspectJ-driven mechanism to declare a field as lazily
    initialised. The aspect intercepts the first read and
    delegates to a supplier. Useful for breaking cyclic references
    between beans whose initialisation order is hard to fix at
    container level.

[IMPORTANT]
====
`LazyFieldAspect` requires AspectJ weaving (load-time or
compile-time). The nutsnbolts POM configures the
`aspectj-maven-plugin` for its own build; consumers who want lazy
fields in their own code must configure weaving separately.
====

[[behavioral-reference]]
== Behavioral Reference

=== Close semantics

`ByteArrayIOStream`, `CircularBufferIOStream`, and their paired
input/output views treat `close()` as a one-way latch. Once closed:

- `asInputStream()` and `asOutputStream()` still return the view
  instances, but subsequent reads and writes throw `IOException`.
- `close()` itself is idempotent; calling it more than once has no
  additional effect.
- Try-with-resources usage is the intended pattern.

=== Context inheritance

`ContextFactory` is backed by `InheritableThreadLocal`. When a new
thread is created from a parent, `childValue` runs and *copies* each
context type's stack into a fresh `LinkedList`. The child has an
independent stack thereafter; push/pop in the child does not affect
the parent.

Any context that refers to mutable state (connection handles,
transactions, open resources) is still *shared by reference* with the
parent — inheritability copies the stacks, not the objects on them.

=== PropertyExpander resolution order

With `SystemPropertyMode.FALLBACK` (the default):

1. Caller map (`expand(expansion, expansionMap)`).
2. System properties (`System.getProperty`).
3. Environment variables (`System.getenv`) if
   `searchSystemEnvironment` is `true`.

With `SystemPropertyMode.OVERRIDE`:

1. System properties.
2. Environment variables (if enabled).
3. Caller map.

Placeholders inside resolved values are expanded recursively; circular
references throw `PropertyExpanderException` rather than looping.

=== Retry scheduling

`Retry.execute` runs the `RetryCall` on a *fresh thread* for each
attempt. The calling thread sleeps between attempts via
`Thread.sleep`, not via scheduled executors, so the method is
appropriate for coarse retries, not for high-frequency polling.

=== Email template caching

`Postman` keeps a `HashMap<SHA256Key, Template>` of compiled FreeMarker
templates, keyed by SHA-256 of the raw body text. Two distinct `Mail`
messages with identical body text share the same compiled template.
The cache has no eviction policy; templates live for the life of the
`Postman` instance. For workloads that generate large numbers of
unique bodies, prefer constructing a fresh `Postman` per campaign and
discarding it afterwards.

=== Resource `null` vs exception

`Resource.getInputStream()` returns `null` when the backing store can
be consulted but does not locate the content (the default convention
across the shipped implementations). It throws `ResourceException`
when the backing store itself errors (permission denied, malformed
URL, failed URL connection). Callers should handle both.

=== GatingClassLoader grace period

`GatingClassLoader` can compare a class's last-modified timestamp
against a `gracePeriodSeconds` tolerance to detect stale definitions
for reloadable extensions. A negative grace period disables the check
entirely; a value of `0` keeps the check enabled with no fresh window,
so the source is consulted on every subsequent `loadClass` call.
Stale detection is not automatic reloading; the loader throws
`StaleClassLoaderException`, and it is up to the host to drop and
recreate the loader.

=== Naive TLS implementations

`NaiveTrustManager`, `NaiveHostNameVerifier`, and
`NaiveSSLSocketFactory` accept every certificate and every hostname.
They exist for controlled test environments (integration tests against
a self-signed fixture). Registering them as JVM-wide defaults silently
disables TLS validation for *every* connection in the process.

[[limitations-and-known-issues]]
== Limitations and Known Issues

Broad scope, narrow depth::
    This package is deliberately broad. Each subpackage solves its
    specific problem and stops. Where a first-class library such as
    Netty, OkHttp, Jackson, Guava, or Bouncy Castle covers the same
    territory with more depth, the expectation is that consumers will
    use that library directly for that surface, not attempt to extend
    the corresponding nutsnbolts subpackage.

Optional dependency failure modes::
    A code path whose optional dependency is missing will fail with
    `NoClassDefFoundError` or `ClassNotFoundException` on first use,
    not at application startup. Read the
    <<installation,Optional Dependency Matrix>> before wiring a new
    subpackage into production.

`Retry` blocks the caller::
    `Retry.execute` runs each attempt on a fresh thread and joins on
    it, but the calling thread is blocked for the full duration of
    all attempts plus all back-off intervals. For high-concurrency
    retry needs, wrap calls in your own executor.

`Postman` template cache has no eviction::
    See <<behavioral-reference,Email template caching>>.

`ssl` naive trust classes are test-only::
    They are shipped for convenience but must never be the default
    in production JVMs. See <<pkg-security-ssl>>.

`inject` is not a DI container::
    `@Service` is a marker. `@LazyField` requires AspectJ weaving.
    Neither substitutes for Spring, CDI, or Guice.

`layout` is toolkit-agnostic by design::
    It does not know about Swing, JavaFX, SWT, or any specific GUI
    widget. Using it in a real application requires writing the
    `ParaboxElement` adapter for your toolkit (the `smallmind-javafx`
    module contains a JavaFX adapter as a reference).

No JPMS `module-info`::
    `smallmind-nutsnbolts` is published as a classpath jar. Consumers
    in a fully modular (JPMS) deployment should expect to use the
    automatic-module-name behaviour rather than an explicit module
    descriptor.

[[dependency-notes]]
== Dependency Notes

* `smallmind-nutsnbolts` sits at the bottom of the dependency graph in
  this repository. Nearly every other SmallMind module depends on it;
  it depends on nothing else in the repository.
* The POM lists all Jakarta, Spring, FreeMarker, Commons-Net, Bouncy
  Castle, ASM, and SnakeYAML dependencies as `optional`. A consumer
  choosing a specific subpackage must add the corresponding runtime
  dependency to its own build.
* The build configures the `smallmind-license-maven-plugin` (for
  dual AGPL/Apache licensing headers) and the `aspectj-maven-plugin`
  (because `LazyFieldAspect` is an AspectJ aspect woven at compile
  time for this jar). A consumer that uses `@LazyField` in its own
  code must configure AspectJ weaving on its side; weaving is not
  transitive through the dependency relationship.
* The shipped Spring XML fragments (`jmx.xml`, `realm.xml`,
  `rmi.xml`) and the command XSD
  (`smallmind-command-1.0.xsd`) are published as classpath resources
  under `org/smallmind/nutsnbolts/`. Consumers import them with
  `classpath:` references.
